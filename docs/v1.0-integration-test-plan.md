## バグ管理

### 重要度分類
- P0: リリースブロッカー（即時修正）
- P1: 重大なバグ（24時間以内）
- P2: 中程度のバグ（リリース前修正）
- P3: 軽微なバグ（v1.1で対応可）

### バグ報告フォーマット
```markdown
**概要**: 
**再現手順**:
**期待される動作**:
**実際の動作**:
**環境**: 
**スクリーンショット**:
**重要度**: P0/P1/P2/P3
```
